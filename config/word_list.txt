# profanity-hush — word list
# =============================================================================
# Merged from two sources:
#   [APF]  AdvancedProfanityFilter defaults (FrostCo/AdvancedProfanityFilter)
#   [orig] Original profanity-hush defaults
#
# ── Match method notation ────────────────────────────────────────────────────
#
#   word       Exact token match, case-insensitive (default).
#              "shit" matches shit, Shit, SHIT.
#
#   =word      Exact token match, case-SENSITIVE.
#              "=dick" matches only lowercase "dick" — not "Dick" (a name).
#              "=Dick" matches only title-case "Dick" — not "dick".
#              Requires transcribe.py to preserve WhisperX's original casing
#              (see note below).
#
#   word*      Starts-with match, case-insensitive.
#              "shit*" matches shit, shitty, shithead, shitface, etc.
#
#   *word*     Contains / substring match, case-insensitive.
#              "*fuck*" matches fuck, fucking, motherfucker, fucktard, etc.
#
# Notation can be combined: "=Shit*" would be a case-sensitive starts-with
# match, though that's unlikely to be useful in practice.
#
# Phrases (entries with spaces) support all of the above on the phrase as a
# whole; * notation is not supported on individual words within a phrase in v1.
#
# ── Case-sensitivity and the transcript ─────────────────────────────────────
#
# WhisperX capitalizes proper nouns and sentence-initial words, so "Dick"
# (a name) and "dick" (profanity) will generally appear with different casing
# in the transcript — making = matching meaningful.
#
# transcribe.py preserves WhisperX's original word casing in the JSON output
# (it does NOT lowercase).  Case-sensitive = entries rely on this behaviour.
# =============================================================================
